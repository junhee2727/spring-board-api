# 프로젝트 규칙

## 중요: 코드 수정 방침
- 내가 명시적으로 요청하지 않은 코드는 절대 수정하지 말 것
- 리팩토링, 코드 정리, 개선 제안을 코드에 직접 적용하지 말 것
- 궁금한 점이 있으면 먼저 질문할 것

## 학습 방식
- 코드를 설명할 때는 왜(why) 중심으로 설명해줄 것
- 더 나은 방법이 있으면 코드 수정 대신 설명으로만 알려줄 것

```
## 대화할 때 좋은 패턴

**설명 요청할 때**
```
이 코드가 어떻게 동작하는지 설명해줘. 코드는 수정하지 마.
```

**막혔을 때**
```
[파일명] 에서 [기능] 구현하려는데 막혔어.
힌트만 줘, 코드 짜주지 말고.
```

**코드 리뷰 받을 때**
```
이 코드 리뷰해줘. 개선점은 설명으로만 알려주고, 직접 수정하지는 마.
```

**직접 구현 요청할 때 (의도적으로)**
```
[기능] 구현해줘. (이건 직접 짜줘도 돼)
